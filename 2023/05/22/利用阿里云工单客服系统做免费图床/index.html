利用阿里云工单客服系统做免费图床

发表于 2023-05-22 更新于 2024-06-26 分类于 折腾

我自己是不用这个图床的，本人有某度！那个还是很稳定的！所以没必要搞很多个，精力耗不起

有需要的可以自行测试，另外根据原作者的意思，key过期了

可以去这里浏览原网页：点击浏览

https://kfupload.alibaba.com/mupload
POST 参数 类型 说明
file 表单 文件的二进制数据
name 参数 随机文件名.jpg
scene 参数 固定为 aeMessageCenterV2ImageRule
支持 JPG、PNG、GIF 格式
文件大小限制为 5MB
无需身份验证
5MB的GIF效果图：

URL：https://ae01.alicdn.com/kf/H5fad35d66dca46108a4898efc0c79f7cT.jpg
URL2：https://sc01.alicdn.com/kf/H5fad35d66dca46108a4898efc0c79f7cT.jpg
google检索了下还有个店铺接口：https://cbu01.alicdn.com/img/ibank/2018/578/980/8997089875_53215101.jpg
回源限速800KB，CDN缓存后边缘节点不限速，10Gbps可跑满

请求类型：POST

URL：https://kfupload.alibaba.com/mupload

文件表单名称：file

参数：scene，值写aeMessageCenterV2ImageRule

参数：name，值写随机.jpg

响应类型，文本，URL值为：$json:url$
要导入的话可以直接复制这个剪贴板导入，记得改成自己的Token就能直接用了：

{
  "Name": "kfupload.alibaba.com",
  "DestinationType": "ImageUploader, FileUploader",
  "RequestType": "POST",
  "RequestURL": "https://kfupload.alibaba.com/mupload",
  "FileFormName": "file",
  "Arguments": {
    "scene": "aeMessageCenterV2ImageRule",
    "name": "itzmx.jpg"
  },
  "ResponseType": "Text",
  "URL": "$json:url$"
}

用来搞视频干啥都行
https://jsbin.com/nofapagalo/edit?html,output
注：没有并发数限制，但是1分钟（可能是10分钟）内上传达到第255个图片会触发ip封禁10分钟，也就是最多只能传254个文件，后面文件就会跳验证码，验证码使用了cookie记录，只需要把cookie拿下来给脚本用就行了
服务器不接受xff，用xff伪造IP服务器不认，代码可以不用写这块没必要。


有cookie后可以绕过验证码出现，可以用无痕浏览器测试，每次都会跳验证码，因为不存在cookie，通过插件提交cookie后就不会跳验证了，或者脚本上写个限制1分钟最多传254个文件
