关键配置解析

配置项 生产环境必要性
数据卷挂载 保证数据持久化，容器重建不丢失数据
资源限制 (--memory) 防止 MongoDB 耗尽宿主机资源，影响其他服务
专用网络 (--network) 隔离数据库流量，提升安全性
密钥文件认证 强制副本集节点间 TLS 通信，防止未授权访问
WiredTiger 缓存限制 避免缓存占用全部内存导致 OOM
日志轮转 防止日志文件无限增长占用磁盘空间
副本集初始化 (--replSet) 为后续扩展高可用集群做准备（单节点也需初始化）

生产环境注意事项

风险项 应对策略
单点故障 扩展为 3 节点副本集：1 Primary + 1 Secondary + 1 Arbiter
数据安全 启用 TLS 加密通信，定期测试备份恢复流程
性能瓶颈 监控慢查询日志 (db.setProfilingLevel(1, 50))，添加合适索引
安全漏洞 定期更新 MongoDB 版本，限制容器仅在内网访问
监控缺失 集成 Prometheus + Grafana 监控关键指标（连接数、内存、Oplog 延迟）
