解决方案

此错误是由于在 MacOS 上交叉编译 Linux 可执行文件时，C 编译器使用了 MacOS 的系统头文件，而 Linux 特有的系统调用 setresgid 和 setresuid 未在 MacOS 头文件中声明。以下是分步修复方法：

1. 安装 Linux 交叉编译工具链

需要安装针对 Linux 的 C 编译器（如 musl-gcc），以正确识别 Linux 系统调用。
